Resolve customer issues via two-way SMS (text messaging) in Amazon Connect

by Abhishek Pandey and Amit Gupta | August 27, 2025 | in Advanced (300), Technical How-to | Permalink |

Introduction

As of 2023, Short Message Service (SMS) has achieved global reach with approximately 5 billion unique users. With 80% of adults using text messages to communicate, it is an essential tool for reaching a broad audience. Amazon Connect’s two-way Short Message Service (SMS) capability enables you to address customer issues via text messaging. This service provides customers with a convenient channel while allowing you to deliver a personalized experience at a lower cost.

Most consumers are familiar with SMS text messages for personal communication purposes. Customers can easily initiate conversations or respond to SMS notifications or appointment reminders to receive assistance from an agent.

In this blog post, we will demonstrate how to use SMS with your Amazon Connect contact center. This solution allows customers to respond to your SMS campaigns. Your agents can receive and reply to customer SMS messages from the same workspace that they use for other channels such as voice, chat, and tasks.

Solution Overview and Architecture

Figure 1: Solution Architecture

The customer sends an SMS message from their mobile device to the SMS-enabled phone number on Amazon Connect.

AWS End User Messaging sends the SMS message as a chat contact to Amazon Connect.

The customer interacts with an Amazon Lex bot over SMS, which collects information through a series of questions. When the customer requests agent assistance, the conversation moves to the next step.

Amazon Connect routes the contact to an agent along with all the information the Amazon Lex bot previously collected. The agent can then begin responding to customer SMS messages.

Amazon Connect routes the agent’s response to the customer through AWS End User Messaging.

AWS End User Messaging sends the response to the customer’s mobile device as an SMS message.

Prerequisites

For this tutorial, you will need the following prerequisites:

An AWS account with administrator access to the following services – Amazon Connect, AWS End User Messaging, Amazon Lex

An Amazon Connect Instance

Amazon Connect two-way SMS

An agent created and associated with the Basic Routing Profile and Basic Queue. Ensure that the Routing Profile has Chat enabled under Channel Availability.

Deploying the Solution

This post walks through a patient contacting the Wellness Clinic to schedule an appointment. The patient sends an SMS message to the clinic’s phone number. Initially, the patient interacts with a bot, then is transferred to a live agent for further assistance.

Step 1: Create an Amazon Lex Bot

Sign in to AWS Management Console and navigate to the Amazon Lex console.

If this is your first bot, select Get Started; otherwise, on the Bots page, under Action, choose Import from the drop-down menu. For more details, refer to Importing and Exporting bots in Amazon Lex V2.

Add the bot name: AppointmentScheduler

Download the bot from this link

Select Browse file, then select the bot you downloaded in step 4

Password is optional

Figure 2: Importing Amazon Lex V2 bot

For IAM permissions, select Create a role with basic Amazon Lex permissions

For COPPA, select No

Leave the idle session timeout as the default value

Figure 3: Amazon Lex V2 Import – IAM and COPPA

Advanced settings are optional and do not require changes. Leave the default value for Warn before overwriting existing bots with the same name.

Figure 4: Amazon Lex V2 Import – Advanced settings

Select Import. You will receive the confirmation: “File successfully imported and created bot AppointmentScheduler”

Step 2: Build the Amazon Lex Bot

Navigate to the Amazon Lex dashboard and select Bots. Click the bot AppointmentScheduler. Under All languages, select English (US). Choose Build.

Figure 5: Build Amazon Lex V2 bot

Step 3: Add the Amazon Lex Bot to your Connect Instance

To open Amazon Connect console, search for Amazon Connect in the search bar and select it.

Figure 6: Search and select Amazon Connect

Select the Amazon Connect instance you want to integrate with your Amazon Lex bot. In the navigation menu, choose Flows.

Figure 7: Configure flows for Amazon Connect

In Amazon Lex, use the dropdown to select the Region of your Amazon Lex bot, then select your bot AppointmentScheduler. Select the bot alias TestBotAlias from the drop-down, then choose + Add Amazon Lex bot.

Note: Create a new alias for production and do not use TestBotAlias.

Figure 8: Add Amazon Lex bot to Amazon Connect Instance

Step 4: Create Amazon Connect Flow

Sign in to your Amazon Connect instance with an account that has permissions for Amazon Connect Flows and Amazon Lex bots. Download the file from the link. From the navigation menu, select Routing and Flows. Click Create Flow. Click the drop-down next to Save, select Import (beta).

Figure 9: Create Connect Flow

Click Choose File, browse to the file you downloaded in step 2, select Import.

Figure 10: Import Connect Flow

After the flow is successfully imported:

Click the Get Customer Input block and select AppointmentScheduler for Lex bot and TestBotAlias for Alias

Select Save

Figure 11: Modify Get Customer Input block

Select the Set Working Queue block and select Basic Queue as shown

Figure 12: Modify working queue block

Select Save, then Publish the flow

Step 5: Associate Amazon Connect Flow to a SMS Phone Number

On the left menu, navigate to Channels → Phone numbers Click the SMS-enabled phone number In Contact Flow/IVR, select SMSBlogFlow from the dropdown and click Save

Figure 13: Associate Connect Flow to SMS phone number

Congratulations! You have successfully added SMS functionality to your Amazon Connect Contact Center.

Below is a screenshot of a patient interacting from their mobile phone via SMS message.

Patient contacting the Wellness Clinic phone number and conversing with the bot.

Figure 14: Patient interaction with Amazon Lex bot over SMS

The patient is then transferred to a live agent to continue receiving help.

Figure 15: Interaction of patient with Amazon Lex bot and live agent via SMS

The agent uses the Amazon Connect CCP (Contact Control Panel) to respond to the patient’s SMS messages.

Figure 16: CCP view showing patient interaction with live agent over SMS

Clean Up

If desired, export the latest version of your Lex bot and save it

If desired, export the SMSBlogFlow Connect flow from Amazon Connect

Disconnect the SMS-enabled phone number from the SMSBlogFlow contact flow

In the left menu, navigate to Channels → Phone numbers

Click the phone number you used to test SMS flow

In Contact Flow/IVR, click X on the far right and Save

Delete the Amazon Lex bot you imported

Archive and delete the Amazon Connect flow you imported

Conclusion

In this blog post, we demonstrated how to enhance SMS messaging capabilities for your Amazon Connect contact center. This solution simplifies customer interaction by enabling direct SMS responses, providing a more efficient and convenient experience for your customers.

As businesses continue to prioritize customer experience, adding SMS support can drive customer satisfaction and reduce operational costs. Get started with setting up SMS messaging for your Amazon Connect Contact Center.

If you need support in implementing this solution, AWS Professional Services is ready to provide guidance and assistance. You may also consider reaching out to Amazon Connect partners worldwide who specialize in deploying and optimizing Amazon Connect solutions.

We recommend the Amazon Connect Administrator Guide, which provides detailed information and guidance to help you better understand Amazon Connect and explore more features.
